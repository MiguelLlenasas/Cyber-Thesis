🔍 Key System Clarifications

  • Tree-Based Classification: Only used during classification phase, NOT during characteristic extraction
  • Data Privacy: Actual passwords are NOT stored; only extracted characteristics are retained
  • Vulnerability Scoring: We identify the most vulnerable attack vector from 3 methods (Rule-Based, Dictionary, Brute-Force)
  • Data Structure: Feature vectors stored as lists of dictionaries for machine learning
  • Tree Algorithm: CART (Classification and Regression Tree) used to train/build decision tree from feature vectors and labeled training data
  • Node Processing: Nodes are utilized during the Tree-Based Classification phase for decision making

Characteristics Extraction (CE) Features

Basic Metrics

  • Length (total number of characters)
  • Lowercase presence (a-z)
  • Uppercase presence (A-Z)
  • Digit presence (0-9)
  • Symbol presence (!@#$%^&*())
  • Character type count

Pattern Analysis

  • Dictionary words detection
  • Leetspeak substitution (a→@, o→0, i→1, e→3, s→$)
  • Capitalization pattern
  • Numeric suffix pattern (password1)
  • Symbol prefix/suffix pattern (@password or password!)
  • Sequential pattern (abc, 123)
  • Repeated character pattern (paaassword)

Aggregated Metrics

  • Rule-based pattern flag
  • Overall vulnerability indicators
  • Feature vector generation

⚙️ Characteristic Extraction Process

1. Input & Dual Version Creation

Example Input: P@ssw0rd2026!!

Original (OG): P@ssw0rd2026!! (for length, character, pattern detection)

Normalized: password2026!! (converted to lowercase, leetspeak replaced)

2. Length Extraction

Count total characters in original password

P@ssw0rd2026!! = 14 characters

3. Character Type Scanning (Single Pass)

Iterate through each character to identify types present

✓ has_lowercase = 1 (found: s, s, w, r, d)
✓ has_uppercase = 1 (found: P)
✓ has_digit = 1 (found: 0, 2, 0, 2, 6)
✓ has_symbol = 1 (found: @, !)
Character Class Count = 4

4. Pattern Detection

Sequential: has_sequence = 0 ✗

Repetition: has_repetition = 1 ✓ (0, 2, 6 might contain)

Leetspeak: has_leetspeak = 1 ✓ (@ for a, 0 for o, 2 for z)

Capitalization: common_capitalization = 1 ✓ (Capital P at start)

Numeric Suffix: numeric_suffix = 1 ✓ (2026 at end)

Symbol Affix: symbol_affix = 1 ✓ (@ at start, !! at end)

5. Dictionary Detection (Using Normalized Version)

Slide a window of 4+ characters against dictionary list

Normalized: password2026!!
Found: "password" in dictionary
dictionary_present = 1 ✓

6. Rule-Based Pattern Aggregation

If ANY of these are true → rule_pattern_present = 1

  • ✓ Leetspeak present
  • ✓ Common capitalization
  • ✓ Numeric suffix
  • ✓ Symbol affix
  • ✓ Sequences or repetition

Result: rule_pattern_present = 1 ✓

7. Feature Vector Output

[
  length = 14,
  has_lowercase = 1,
  has_uppercase = 1,
  has_digit = 1,
  has_symbol = 1,
  character_class_count = 4,
  dictionary_present = 1,
  has_leetspeak = 1,
  common_capitalization = 1,
  numeric_suffix = 1,
  symbol_affix = 1,
  has_sequence = 0,
  has_repetition = 1,
  rule_pattern_present = 1
]

⚠️ Data Privacy: Both password versions (OG and Normalized) are deleted after characteristic extraction due to Data Privacy Act compliance.

🌳 Tree-Based Classification (TBC) Phase

The feature vector enters the decision tree for classification based on CART algorithm-trained nodes.

Decision Tree Logic

Node 1: Dictionary present?

YES → Go to Rule-Based or Dictionary path (check additional features)

NO → Check character class count

Node 2 (If No Dictionary): Character class count < 3?

YES → Brute-Force vulnerability

NO → Check for sequences

Node 3 (If Char Class ≥ 3): Sequence present?

YES → Rule-Based vulnerability

NO → Brute-Force vulnerability

Example: P@ssw0rd2026!!

1

Check: Dictionary present?

YES (contains "password")

2

Check: Has leetspeak?

YES (@ for a, 0 for o)

3

Check: Additional pattern checks...

Multiple rule-based patterns detected

✓

Leaf Node Classification:

RULE-BASED Attack Vulnerability

Why RULE-BASED? The password contains dictionary substrings combined with predictable transformations (leetspeak), making it vulnerable to rule-based dictionary attacks that apply common transformation rules.

⚔️ Three Vulnerability Classification Types

Rule-Based Attack

🔴 Most Vulnerable

Characteristics: Dictionary words with predictable transformations

  • Contains dictionary substrings
  • Leetspeak substitutions detected
  • Common capitalization patterns
  • Numeric suffixes or symbol affixes

Attack Method: Dictionary attack combined with transformation rules (leetspeak converters, case variations)

Dictionary Attack

🟠 Moderately Vulnerable

Characteristics: Contains existing dictionary words

  • Password contains common words
  • No complex transformations
  • Basic structure

Attack Method: Direct dictionary lookup against common password lists

Brute-Force Resistant

🟢 Most Resistant

Characteristics: No patterns, no sequences, no dictionary words

  • Random character combinations
  • High complexity
  • No identifiable patterns

Attack Method: Requires exhaustive character-by-character guessing (computationally expensive)

📌 Note: Brute-Force is the MOST FAVORABLE for users (hardest to crack). Rule-Based is the LEAST FAVORABLE (easiest to crack). Dictionary falls in between.
